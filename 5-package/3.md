# 🟢 Constants in Go

### 📌 What are Constants?

* Constants are declared using the `const` keyword
* Their value **cannot change** after declaration
* Can be:

  * Numbers
  * Strings
  * Booleans
  * Characters

👉 Unlike variables:

* ❌ Cannot use `:=` syntax
* ❌ Cannot be reassigned

---

## 🧠 Typed vs Untyped Constants

* **Typed constant** → has a fixed type
* **Untyped constant** → flexible, adapts to context

---

## 📁 Example

### 1. `constant.go`

```go
package main

import (
	"fmt"

	"github.com/krishnakumarkp/go-training/package/company"
)

func main() {
	company.PrintCompany()
	company.PrintDepartment()

	// Typed constant
	const a int64 = 8

	// Untyped constant
	const b = 8

	var c int64

	c = a // ✅ same type
	c = b // ✅ untyped constant adapts

	fmt.Println(c)

	// Problem with typed constants
	var growth float32
	// var growth float64

	growth = company.GrowthPercentage

	fmt.Println("growth percentage =", growth)
}
```

---

### 2. `company/company.go`

```go
package company

const GrowthPercentage float64 = 6.1415926535897932384
```

---

## ⚠️ Key Learning

### 🔴 Typed Constant Issue

```go
var growth float32
growth = company.GrowthPercentage // ❌ may cause precision/type issue
```

👉 Because:

* `GrowthPercentage` is **typed as float64**
* Assigning to `float32` may require explicit conversion

---

### ✅ Better Approach (Untyped Constant)

```go
const GrowthPercentage = 6.1415926535897932384
```

👉 Now:

* It adapts automatically
* Works with both `float32` and `float64`

---

## 🧠 Summary

| Concept             | Behavior               |
| ------------------- | ---------------------- |
| `const a int64 = 8` | Typed constant         |
| `const b = 8`       | Untyped constant       |
| Untyped constants   | Flexible, auto-convert |
| Typed constants     | Strict, fixed type     |
| `:=` with const     | ❌ Not allowed          |

---

## 🚀 Best Practice

👉 Prefer **untyped constants** unless you specifically need a fixed type.
